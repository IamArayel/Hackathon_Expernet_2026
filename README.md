### Gestion de la structure du projet :
- 1 carte sur le Github Project = 1 branche = 1 Pull Request
- Des noms de branche en français : **`<type>-ticket(-<description>)`**
  - _type_ (requis) : C'est le type de contribution auquel le code se rapporte :

    - feature (nouvelle fonctionnalité)
    - bugfix (correction non urgente)
    - hotfix (correction urgente prod)
    - security (faille de sécurité)
    - test (tests)
    - refactor (structure interne)
    - chore (maintenance / tooling)
  - _ticket_ (numéro de l'issue Github)
  - _description_ : courte description du contexte abordé
- Des messages de commit en français, respectant une convention claire et approuvée : https://www.conventionalcommits.org/fr/v1.0.0/#summary
- **1 review obligatoire** avant de pouvoir merger sur la branche `main`
- Une carte dans le Github Project complétée d'après le template fourni.
